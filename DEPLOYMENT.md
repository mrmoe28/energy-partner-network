# Deployment Guide

This project is a Vite-built SPA served by a small Node runtime that also handles the lead and contact APIs.

## Prerequisites

- Node.js (v18 or higher)
- Docker
- Supabase CLI (for local development with Ekobase)

## Local Development

### Starting the Supabase Backend

1. Navigate to the project directory
2. Run `supabase start` to start the local Supabase instance
3. The services will be available at:
   - Studio: http://127.0.0.1:54333
   - API: http://127.0.0.1:54331
   - Database: postgresql://postgres:postgres@127.0.0.1:54332/postgres

For more details about the Supabase integration, see [SUPABASE_INTEGRATION.md](SUPABASE_INTEGRATION.md).

### Starting the Development Server

1. Install dependencies:
   ```bash
   npm install
   ```

2. Run development server:
   ```bash
   npm run dev
   ```

3. Build for production:
   ```bash
   npm run build
   ```

4. The production build will be generated in the `dist` directory.

### Running The App Server

The runtime server serves `dist/` and exposes:

- `POST /api/lead-signups`
- `POST /api/contact-submissions`
- `GET /api/lead-signups`
- `GET /api/contact-submissions`
- `GET /admin/leads`

Start it locally after building:

```bash
npm run start
```

## Docker Deployment

The project includes a Dockerfile for containerized deployment:

1. Build the Docker image:
   ```bash
   docker build -t energy-partner-network .
   ```

2. Run the container:
   ```bash
   docker run -d -p 3000:3000 energy-partner-network
   ```

## Coolify Deployment

Production URL:

- `https://energy-partner-network.lock28.com`

To deploy to Coolify:

1. Create a new project in Coolify
2. Link your GitHub repository
3. Choose a Dockerfile-based deployment so Coolify builds and runs the bundled production container.
4. Expose port `3000` if Coolify does not detect it automatically.
5. Provide these runtime environment variables:
   - `DATABASE_URL`: PostgreSQL connection string for Ekobase
   - `ADMIN_USERNAME`: Basic auth username for the admin inbox
   - `ADMIN_PASSWORD`: Basic auth password for the admin inbox
6. Coolify will automatically build and deploy your site

## Environment Variables

Runtime environment variables:

- `DATABASE_URL`: Required for the server to connect to Ekobase
- `ADMIN_USERNAME`: Optional, defaults to `admin`
- `ADMIN_PASSWORD`: Optional, defaults to `admin`

The browser build still only depends on the static assets generated by Vite.

## Testing

Run tests with:
```bash
npm run test
```

Run tests in watch mode:
```bash
npm run test:watch
```

Generate coverage report:
```bash
npm run test:coverage
```
